TGR Documentation Index

The TGR documentation system consists of ten documents organized from theoretical foundations to engineering implementation. Each document has a clearly defined role and addresses one independent problem.

The documents are connected through interface constraints rather than direct dependencies. Any document may be replaced independently, provided that it preserves the interface constraints with adjacent layers.

Updating Principle

Theory documents evolve through full audit. Engineering specifications remain compatible with the theory.

Signal Taxonomy is open to new Signal types. APIs remain stable whenever possible. Engine implementations may evolve freely as long as API contracts are preserved.

This index serves only as a navigation document. It does not define any concept. The authoritative definitions are contained in the individual documents.
